# Systematic prompt template

Use these headings in order:

1. Mission and scope
2. Authority and rights boundary
3. Source-of-truth order
4. Visual north star
5. Non-negotiable invariants
6. Design tokens
7. Layout and responsive behavior
8. Component grammar
9. Content density and imagery
10. Interaction and motion
11. Accessibility and performance
12. Negative constraints
13. Unknown-handling rules
14. Acceptance checklist
15. Iteration protocol

## Writing rules

- Write executable constraints, not mood-board adjectives alone.
- Put the most consequential 5–8 rules in non-negotiable invariants.
- Include token values and role relationships.
- Separate fixed facts from responsive rules and inferred fallbacks.
- Tell the downstream model what not to copy and what not to invent.
- Require semantic, editable UI; prohibit using a screenshot as the implementation.
- Explain how to evaluate and revise the first result.
