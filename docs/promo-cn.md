## 朋友圈短版（推荐）

最近把一个我自己一直很想要的工具开源了：**UItoPrompt**。

我们经常看到一个喜欢的网页或截图，然后丢给 AI 说：“照这个风格做一个。”但“极简、现代、克制、有呼吸感”这类形容词，通常不足以让模型真正复现它的排版、色彩、留白和组件节奏。

UItoPrompt 做的是中间最缺的那一层：把公开网页、UI 截图，甚至海报、摄影和杂志版式，拆成可检查的视觉证据，再整理成 **StyleSpec + 15 章 Systematic Prompt + CSS 变量 + 证据报告 + 可复用 Skill**。

它最大的不同，是不会把猜测伪装成事实。每条重要规则都会标记为：直接观察、程序测量、基于证据推断、从非 UI 视觉转译、用户确认，或者仍然未知。

如果输入只是一张桌面截图，它不会假装知道原网页的 DOM、字体文件、移动端断点、Hover 状态和完整动效；证据不足的部分会明确保留为未知，并告诉后续模型应该怎样保守处理。

默认模式只提炼视觉语言，不复制来源品牌、文案、Logo、图片和其他受保护资产。生成的 Skill 可以继续交给 Codex，也可以作为 Claude Code、Cursor、v0 等工具的透明工作流参考，让不同模型长期复用同一套设计规则。

目前源码已经用 MIT License 正式开源。在线版 `uitoprompt.com` 会在出站网络隔离和公网限流完成后再部署；现在可以先在本地完整运行和审查。

GitHub：**https://github.com/derrickgong87/ui-to-prompt**

## 朋友圈长版

最近把一个我自己一直很想要的工具做完第一版，并正式开源了：**UItoPrompt**。

举个非常具体的例子。

你看到一个网页，喜欢它的排版、颜色、留白和组件节奏。以前通常是截张图，丢给大模型说：“照这个风格做一个。”模型也许会写出一段听起来很专业的话——极简、现代、克制、富有呼吸感——但真正开始生成时，结果往往还是熟悉的渐变、三栏卡片和大圆角。

原因很简单：**“形容得像”，不等于“规则提取得对”。**

UItoPrompt 想解决的，就是这中间缺失的一层：把一个公开网页、一张 UI 截图，甚至海报、杂志、摄影或插画，转换成大模型真正可以执行、检查和长期复用的设计规则。

使用方式很直接：粘贴一个公开网页，或者上传一张界面截图。UItoPrompt 会把参考内容拆成色彩角色、字体层级、间距节奏、网格与布局、圆角与阴影、组件语法、内容密度、响应式规则、交互与动效，以及明确的负面约束。

最后输出的不是一段“氛围描述”，而是一套完整文件：

- 机器可读、可版本化的 StyleSpec；
- 固定 15 个章节的 Systematic Prompt；
- 可以立刻交给模型的精简 Prompt；
- CSS 设计变量；
- 证据与不确定性报告；
- 可以在后续项目复用的 AI Style Skill。

它和普通“截图转 Prompt”工具最大的区别，是会把结论分清楚：

- 哪些是画面里直接看到的；
- 哪些是浏览器或程序实际测量出来的；
- 哪些只是多条线索支持的推断；
- 哪些是从海报、照片或插画转译成的 UI 规则；
- 哪些来自用户确认；
- 哪些信息目前根本不知道。

比如只有一张桌面截图时，工具不会假装自己知道原网页的 DOM、真实字体文件、移动端断点、Hover 状态和完整动效。这些内容会被明确标成“未知”，并告诉后续模型应该采用什么保守方案、还需要补充什么证据。

对我来说，这种诚实比一个虚假的“98% 还原度”更重要。

它适合很多场景：

- 做新产品时，把喜欢的参考整理成一套不依赖原品牌的原创视觉语言；
- 重构旧项目时，把现有界面提炼成团队共享的设计规则；
- 设计师把参考图交给工程 Agent，不必每次重新解释审美；
- 开发者在多个项目、多个模型之间复用同一套 Style Skill；
- 从杂志、海报或摄影里只提取节奏、对比、色彩和形状，再转译成原创 Web UI；
- 为自己拥有的产品做一次“设计系统考古”，把隐含规则变成显式规范。

它内部的工作流是：

**网页 / 图片 → 视觉证据 → StyleSpec → Systematic Prompt → 可复用 Skill**

网页模式会尽量读取真实结构、计算样式、字体、可见几何信息、资源和视口事实；图片模式会在本地分析像素、颜色、构图与可见层级。两种来源最后进入同一个结构化规范，再由确定性编译器生成固定 15 章 Prompt。

项目默认使用“风格提炼”模式：保留视觉语法，但移除来源品牌、文案、Logo、图片、专有图标和受保护资产。只有在页面属于自己或已经获得授权时，才应该使用更忠实的“授权重建”模式。

第一版已经完成本地产品、核心 Skill、像素分析、URL 安全取证、Systematic Prompt 编译器、完整样例和文件导出。桌面端、手机端、本地图片和真实 URL 流程都做过浏览器验收；当前自动化验证包含 60 项 Node 测试、8 项 Python Skill 测试和 1 项 PowerShell 安全打包测试。
